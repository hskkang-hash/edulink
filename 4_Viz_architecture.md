## 3. 정산 흐름 (Settlement Workflow)

### 3.1 자동 정산 프로세스

```

1️⃣ 어제자 완료된 수업 조회
   │
   ├──> SELECT sessions
   │    WHERE DATE(actual_end) = YESTERDAY
   │    AND status = 'completed'
   │
   └──> 수업 45개 발견
   
2️⃣ 강사별 수익 합계 계산
   │
   ├──> 강사 A: 5개 수업 × 40,000원 = 200,000원
   ├──> 강사 B: 3개 수업 × 50,000원 = 150,000원
   ├──> 강사 C: 4개 수업 × 35,000원 = 140,000원
```

### 3.2 MLM 보상 계산 흐름

```
월말 (1일)

1️⃣ 전체 강사 월 매출액(PV) 조회
   │
   └──> SELECT instructor_id, SUM(revenue) as pv
        FROM sessions
        WHERE MONTH(completed_at) = CURRENT_MONTH
        GROUP BY instructor_id

2️⃣ 레벨별 분류
   │
   ├──> Level 1: PV ≥ 200,000 (245명)
   ├──> Level 2: PV ≥ 5,000,000 (45명)
   └──> Level 3: PV ≥ 10,000,000 (8명)

3️⃣ 레벨별 보상율 적용
   │
   ├──> Level 1: 3% = 3,200,000원 (기본 수수료에서 배분)
   ├──> Level 2: 6% = 6,800,000원 (다운라인 실적 + 본인)
   └──> Level 3: 21% = 4,100,000원 + Leadership Bonus 2,150,000원
                        = 6,250,000원

4️⃣ MLM 규제 확인 (방판법 준수)
   │
   ├──> 총 수익: 50,000,000원 (Γ)
   ├──> 총 보상: 16,250,000원 (C)
   │
   ├──> 비율: C/Γ = 16,250,000 / 50,000,000 = 32.5%
   │
   ├──> 규제: C ≤ 0.35 × Γ = 17,500,000원
   │
   └──> 결과:  32.5% < 35% → ✅ SAFE
               (1,250,000원 여유)

5️⃣ 조정 필요?
   │
   ├──> NO (32.5% < 35%)
   │    → 그대로 지급
   │
   └──> YES (만약 36%)
        → Level 3 보상 자동 감소
        예) 21% → 15% 조정 → 비율 34% 미만으로 강제

6️⃣ 개별 강사 최종 정산액 계산
   │
   ├──> 강사 A (Level 2):
   │    수익: 1,000,000원
   │    보상: 1,000,000 × 0.06 = 60,000원
   │    지급: (1,000,000 - 33,000 원천징수) + 60,000 = 1,027,000원
   │
   ├──> 강사 B (Level 3):
   │    수익: 2,000,000원
   │    보상: 2,000,000 × 0.21 = 420,000원
   │    지급: (2,000,000 - 66,000) + 420,000 = 2,354,000원
   │
   └──> 강사 C (Level 1):
        수익: 500,000원
        보상: 500,000 × 0.03 = 15,000원
        지급: (500,000 - 16,500) + 15,000 = 498,500원

7️⃣ DB 저장 및 경고
   │
   ├──> INSERT mlm_compensation
   │    (instructor_id, level, pv, bonus_rate, bonus_amount, ...)
   │
   └──> IF mlm_ratio > 34% THEN
        ALERT: "MLM 규제 경고: 비율 34.2% 접근"
        (관리자에게 통보)
```
