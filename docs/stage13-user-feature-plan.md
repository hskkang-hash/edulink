# EduLink Stage 13+ 사용자 기능 중심 개발계획

Last Updated: 2026-03-23

## 목표
- 설계 중심 완료(Stage 1-12) 이후, 실제 사용자가 업무를 끝까지 수행할 수 있는 기능 완성에 집중한다.
- 역할별 핵심 시나리오를 End-to-End로 연결하고, 성공/실패/예외 케이스까지 검증한다.

## Stage 13. 인증/권한 사용자 경험 완성
- 사용자 기능 목표:
- 회원가입, 로그인, 로그아웃, 비밀번호 변경/재설정까지 사용자가 스스로 계정 복구 가능해야 함
- 역할별 진입 화면과 메뉴 제한이 일관되게 동작해야 함
- 개발 항목:
- 토큰 만료/에러 메시지 표준화
- 권한 없는 화면 진입 시 안전한 리다이렉트
- 세션 유지 및 재로그인 UX 정리
- 완료 기준:
- 강사/학습자/운영자 각 계정으로 로그인 후 권한 범위 내 화면만 접근 가능
- 비밀번호 재설정 플로우 단일 세션 테스트 통과

## Stage 14. 강사 핵심 플로우 E2E
- 사용자 기능 목표:
- 공고 탐색 -> 지원 -> 상태 확인 -> 정산/세무 확인까지 한 흐름으로 이어짐
- 개발 항목:
- 공고 필터/정렬/상세 보기 UX 고도화
- 지원 상태 실시간 반영(대기/승인/거절)
- 마이페이지형 최근 활동/지원 내역 요약
- 완료 기준:
- 강사 계정 기준 대표 시나리오 3개 E2E 성공

## Stage 15. 기관(학교) 운영 플로우 E2E
- 사용자 기능 목표:
- 공고 등록 -> 지원자 심사 -> 배정 -> 정산 승인까지 운영자가 완료 가능
- 개발 항목:
- 공고 작성/수정 모드 안정화
- 지원자 비교/상태 변경 액션 강화
- 정산 승인 내역 및 이력 표시
- 완료 기준:
- 기관 운영자 대표 시나리오 3개 E2E 성공

## Stage 16. 교육청 거버넌스 플로우 E2E
- 사용자 기능 목표:
- 예산/커버리지 모니터링과 정책 브리핑 데이터 확인 가능
- 개발 항목:
- 지역 단위 필터 프리셋 고도화
- KPI/활동 피드 기반 진단 뷰 정리
- 브리핑 패널 데이터 최신화 액션
- 완료 기준:
- 교육청 사용자 기준 의사결정 시나리오 2개 이상 성공

## Stage 17. 플랫폼 운영/컴플라이언스 고도화
- 사용자 기능 목표:
- 운영자가 지급 무결성, 규칙 감사, 이상징후 확인을 빠르게 수행
- 개발 항목:
- 규칙 시뮬레이션 UX 개선(결과 비교/내보내기)
- 감사 로그 탐색/필터 강화
- 위험 등급 표시 정확도 점검
- 완료 기준:
- 운영자 시나리오(요약 조회, 시뮬레이션, 감사 확인) E2E 성공

## Stage 18. 품질/안정화 릴리스 준비
- 사용자 기능 목표:
- 실제 사용자 테스트에서 막힘 없는 서비스 경험 제공
- 개발 항목:
- 다국어 카피 정리(EN/KO)
- 오류 메시지/로딩/빈상태 UX 통일
- 회귀 점검 체크리스트 및 릴리스 노트 작성
- 완료 기준:
- 핵심 플로우 회귀 테스트 통과
- 릴리스 체크리스트 완료

## 공통 운영 규칙
- 각 Stage는 완료 즉시 [docs/development-progress.md](docs/development-progress.md)에 체크 표시
- 기능 추가 시 반드시 사용자 행동 기준(시작 -> 처리 -> 완료)으로 테스트
- 프론트 변경은 역할별 권한/상태 동기화 회귀 점검 포함
