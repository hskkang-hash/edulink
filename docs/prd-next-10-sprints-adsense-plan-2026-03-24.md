# EduLink PRD 기반 다음 단계 개발계획 (10 Sprint)

작성일: 2026-03-24  
기준 문서: EDULINK_PRD.md, DEV_PLAN.md, docs/development-progress.md, backend/app_jwt.py, frontend/index.html

## 1) 현재 상태 진단 (PRD 기준)

### 1.1 확인된 강점
- 단일 백엔드 엔트리포인트가 사실상 backend/app_jwt.py로 정리됨
- 인증, 공고, 지원, 대시보드, 세무(Mock), 네트워크(MLM), 관리자/교육청 API가 이미 다수 구현됨
- 강사 프로필, 세션, 리뷰, SOS, 계약, 에스크로 API까지 엔드포인트가 존재함
- 테스트 파일이 역할별/플로우별로 분리되어 기본 회귀 기반이 있음

### 1.2 PRD 대비 주요 공백
- 사용자 편의성: 단일 대형 frontend/index.html 기반으로 정보 구조가 복잡하고, 실제 사용자별 "짧은 완료 경로" UX가 약함
- 기능 완결성: API 존재 대비 화면 완성도 및 예외/에러 상태 UX가 불균등함
- 신뢰/정책: 애드센스 심사 필수 요소(정책 페이지, 연락처, 사이트 구조, 콘텐츠 허브)가 명시적으로 보이지 않음
- 운영 품질: 문서별 완료 체크와 실제 코드 상태 간 불일치 가능성(완료 기준 재정의 필요)

### 1.3 이번 계획의 목표
- "사용자 개별 기능"을 완결형으로 고도화: 강사, 기관, 운영자, 교육청 각각 핵심 과업을 끝까지 수행 가능하게 함
- 10개 Sprint로 분할하고, 각 Sprint에 완료 화면(DoD UI)을 명시함
- 애드센스 심사 통과 가능성을 높이는 제품/정책/콘텐츠/기술 패키지를 병행 구축함

## 2) 10 Sprint 실행계획 (사용자 편의성 + 완결성 중심)

## Sprint 1. 사용자 여정 재정의 + IA 정리
목표:
- 역할별 첫 진입 후 3클릭 이내 핵심 행동(강사: 공고지원, 기관: 공고등록, 운영자: 심사, 교육청: 리포트조회)

개발 항목:
- 역할별 홈 진입 라우팅 정리
- 상단/사이드 메뉴를 역할별 최소 메뉴로 재구성
- 주요 CTA(다음 행동) 고정 영역 적용

완료 화면(DoD UI):
- "역할별 홈"에서 첫 CTA가 화면 상단 1스크롤 내 노출
- 잘못된 권한 화면 접근 시 안내 + 안전 리다이렉트

## Sprint 2. 강사 온보딩 완결 (진입장벽 최소화)
목표:
- 강사 가입 후 프로필 제출/심사 상태 확인까지 단일 플로우 완성

개발 항목:
- 온보딩 단계별 저장(임시저장/재개)
- 필수 동의(원천징수/신원조회) UX 개선
- 반려 사유 노출 + 재제출 동선 제공

완료 화면(DoD UI):
- 온보딩 진행률 바(예: 4/4), 제출 완료 배너, 심사 상태 카드

## Sprint 3. 강사 공고 탐색/지원 완결
목표:
- 강사 관점에서 "검색 -> 비교 -> 지원 -> 추적"이 1개 화면군에서 닫힘

개발 항목:
- 필터 프리셋(내 과목/내 지역/마감임박)
- 공고 상세 비교 카드(시급, 위치, 기관평점, 일정)
- 지원 후 상태 변화를 타임라인으로 제공

완료 화면(DoD UI):
- 공고 리스트 + 상세 + 지원상태 타임라인이 끊김 없이 연결됨

## Sprint 4. 수업 세션(체크인/완료/일지) 완결
목표:
- 강사가 실제 수업 단위로 체크인부터 완료보고까지 끝낼 수 있음

개발 항목:
- 체크인 허용 시간/거리 검증 메시지 명확화
- 수업 완료 폼(일지/증빙) 단순화
- 실패 재시도와 네트워크 오류 복구 UX

완료 화면(DoD UI):
- "오늘 수업" 카드에서 체크인 버튼 상태가 실시간 변경됨
- 완료 후 정산 예정 금액이 즉시 표시됨

## Sprint 5. 정산/세무 사용자 완결
목표:
- 강사가 월별 순수령/원천징수/환급흐름을 스스로 이해하고 실행 가능

개발 항목:
- 월별 정산 브레이크다운 시각화
- 환급 조회/신고/진행상태를 1개 허브로 통합
- CSV/명세 다운로드 개선

완료 화면(DoD UI):
- 월별 정산 요약 카드 + 환급 진행 타임라인 + 다운로드 액션

## Sprint 6. 기관 운영 플로우 완결
목표:
- 기관이 공고 생성부터 지원자 심사, 배정, 정산 확인까지 완료 가능

개발 항목:
- 공고 템플릿(반복 공고 빠른 생성)
- 지원자 비교표(경력/리뷰/가능시간)
- 승인/거절 일괄처리 및 사유 템플릿

완료 화면(DoD UI):
- "내 공고"에서 지원자 비교표와 심사 액션이 한 화면에서 완료됨

## Sprint 7. 운영자 승인/컴플라이언스 완결
목표:
- 운영자가 프로필 승인, MLM 35% 준수 모니터링, 감사로그 확인을 신속 수행

개발 항목:
- 심사 큐 우선순위(긴급/대기시간 기준)
- 규칙 시뮬레이션 결과 비교뷰
- 감사로그 검색/필터/내보내기

완료 화면(DoD UI):
- "컴플라이언스 대시보드"에 위험신호, 조치 버튼, 변경이력이 동시 노출

## Sprint 8. 교육청 거버넌스 리포트 완결
목표:
- 교육청 사용자가 지역/기관 단위 KPI를 확인하고 리포트를 생성

개발 항목:
- 지역 비교 필터 개선
- 기관 상세 Drill-down
- 리포트 생성(CSV 우선, PDF 차순위)

완료 화면(DoD UI):
- KPI 보드 + 지역비교 차트 + 리포트 다운로드 버튼

## Sprint 9. 신뢰 기능 고도화 (SOS/리뷰/계약/에스크로)
목표:
- PRD 차별화 기능을 사용자에게 "보이는 기능"으로 전환

개발 항목:
- SOS 실시간 알림(SSE 우선)
- 리뷰 신뢰도(검증 리뷰 배지)
- 계약 상태/서명 추적 UI
- 에스크로 결제 상태(보관/해제/환불) 타임라인

완료 화면(DoD UI):
- 긴급 매칭 수락, 계약 서명, 에스크로 상태가 각각 사용자 대시보드에서 확인 가능

## Sprint 10. 릴리스 + 애드센스 심사 대응 패키지
목표:
- 사용성 QA와 정책/콘텐츠/기술 요건을 정리해 심사 준비 완료

개발 항목:
- 오류/빈상태/로딩 UX 통합
- 핵심 페이지 성능/접근성 점검
- 애드센스 심사 필수 페이지 및 사이트 구조 완성

완료 화면(DoD UI):
- 정책 페이지 링크가 푸터/헤더에 상시 노출
- 콘텐츠 허브와 문의 페이지가 탐색 가능

## 4) 애드센스 심사 통과 준비안 (실무 체크리스트)

중요:
- 애드센스 "통과 보장"은 불가능하지만, 아래 항목을 채우면 통과 확률을 크게 높일 수 있음

## 4.1 필수 페이지 (반드시 공개 URL로 제공)
- 개인정보처리방침
- 이용약관
- 서비스 소개(Who we are)
- 문의하기(연락 가능한 이메일/폼)
- 쿠키/광고 고지(개인화 광고 여부 포함)

## 4.2 콘텐츠 품질 요건
- 색인 가능한 고유 콘텐츠를 최소 20개 이상 확보(교육 운영 팁, 강사 가이드, 기관 운영 가이드)
- 각 글 800자 이상, 중복/얕은 문서 회피
- 카테고리 구조: 강사, 기관, 교육청, 세무/정산, 공지
- 작성자/수정일/출처 표기

## 4.3 UX/기술 요건
- 글로벌 내비게이션 + 푸터 링크 완비
- 모바일 가독성(폰트/간격/터치 타깃)
- 404/500 커스텀 페이지
- 사이트맵(sitemap.xml), robots.txt 제공
- 페이지 속도 개선(LCP/CLS 기본 점검)

## 4.4 광고 정책 요건
- 금지 콘텐츠/민감 콘텐츠 필터 정책 수립
- 과도한 광고 배치 금지(상단 광고 과밀 금지)
- 자동 리디렉션/팝업/강제 클릭 유도 금지
- 무효 트래픽 방지(자체 클릭/봇) 운영 가이드 작성

## 4.5 구현 권장 순서 (2주)
- D1-D2: 정책 페이지 5종 작성/배포
- D3-D6: 콘텐츠 허브 + 초기 문서 20개 게시
- D7-D9: 내비/푸터/사이트맵/robots/오류페이지 반영
- D10-D12: 광고 슬롯 배치 최소화 + Core UX 점검
- D13-D14: 최종 QA 후 애드센스 신청
