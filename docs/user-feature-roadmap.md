## 0. 현재 상태 진단

### 역할별 현재 구현 수준

| 역할 | 코드 키 | 현재 접근 화면 | 실제 동작 가능 기능 | 갭 |
|------|--------|--------------|-------------------|----|
| 강사 | `instructor` | dashboard, postings, applications, tax, network, createPosting, mypage | 공고 목록 조회, 공고 등록/수정/삭제, 세무 견적, 네트워크 보너스 | 지원하기, 지원 상태 추적, 정산 내역, SOS 알림 |
| 학습자(기관) | `student` | dashboard, postings, applications, mypage | 공고 목록 조회, 지원하기 | 지원 상태 추적, 배정 확인, 정산 승인 |
| 운영자 | `admin` | dashboard, applications, adminInsights, mypage | 지원 목록 조회, 상태 변경 | 회원 관리, 규칙 시뮬레이션, 감사 로그 |
| 슈퍼운영자 | `super_admin` | admin과 동일 | admin과 동일 | 동일 |

### 백엔드 현재 API 엔드포인트 (app_jwt.py 기준)

- ✅ Auth: register, login, me, change-password, request-reset, confirm-reset
- ✅ Postings: GET/POST /postings, GET/PUT/DELETE /postings/:id
- ✅ Applications: GET/POST /applications, PUT /applications/:id/status
- ✅ Tax: GET/POST tax-profile, GET/POST tax-claims, GET tax-events
- ✅ Network: GET bonuses, GET compliance, POST sales
- ✅ Admin: GET compliance-summary, POST simulate, GET rule-audits
- ❌ Dashboard: stats API 없음 (mock)
- ❌ 긴급 매칭 (SOS): 없음
- ❌ 일정 관리: 없음
- ❌ 회원 관리: 없음

| 현행 코드값 | 현행 제목 | 목표 코드값 | 목표 제목 | 대상 사용자 |

## Sprint A. 인증 + 역할 UX 완성 (1주)

**목표**: 모든 역할이 로그인부터 첫 화면까지 막힘 없이 진입 가능

### A-1. 역할 선택 회원가입 개선
- [ ] 회원가입 role 드롭다운 → 4가지 역할 완전 표시 (`instructor` / `institution` / `district` / `admin`)
- [ ] 역할별 설명 문구 추가 (강사 / 기관 운영자 / 교육청 / 플랫폼 운영자)
- [ ] 가입 완료 후 역할별 기본 화면으로 자동 이동

### A-2. 역할별 네비게이션 메뉴 차별화
- [ ] `instructor` 메뉴: 공고 탐색 · 내 지원 · 정산/세무 · 네트워크 · 마이페이지
- [ ] `institution` 메뉴: 공고 관리 · 지원자 관리 · 정산 현황 · 마이페이지
- [ ] `district` 메뉴: 대시보드(통계) · 예산 현황 · 기관 리스트 · 마이페이지
- [ ] `admin` 메뉴: 회원 관리 · 지원 현황 · 컴플라이언스 · 규칙 시뮬레이션 · 마이페이지

### A-3. 세션 보호
- [ ] 토큰 만료 시 로그인 화면으로 자동 리다이렉트 + 안내 메시지
- [ ] 권한 없는 화면 접근 시 기본 화면으로 안전 이동

### 완료 기준
- 각 역할 계정으로 로그인 → 허용 화면만 메뉴에 표시됨
- 잘못된 URL/화면 접근 시 기본 화면으로 리다이렉트됨

## Sprint B. 강사 기능 완성 (2주)

**목표**: 강사가 공고를 찾고 → 지원하고 → 상태를 확인하고 → 수입을 파악할 수 있음

### B-1. 공고 탐색 (postingsSection)
- [ ] **공고 카드 디자인 개선**: 제목 / 과목 / 지역 / 시급 / 긴급 배지 표시
- [ ] **지원하기 버튼**: 강사 역할에서만 표시, 이미 지원 시 "지원 완료" 비활성화 상태로 표시
- [ ] **상세 보기 모달**: 공고 상세 (기관 정보, 수업 시간, 자격요건, 지원 마감일)
- [ ] **필터 강화**: 과목, 지역, 최소 시급, 긴급 공고 여부
- [ ] **정렬**: 최신순 / 시급 높은순 / 마감 임박순

**백엔드**: `GET /postings` → 필터 파라미터 `urgent`, `sort_by` 추가 / `POST /applications` 중복 지원 방지 로직

### B-2. 내 지원 현황 (applicationsSection - 강사 뷰)
- [ ] 강사 전용 지원 내역 목록: 공고명 / 지원일 / 상태(대기/승인/거절)
- [ ] 상태별 탭 필터 (전체 / 대기중 / 승인됨 / 거절됨)
- [ ] 승인된 지원 → "수업 일정 확인" 링크 표시
- [ ] 거절된 지원 → 사유 표시 (있을 경우)

**백엔드**: `GET /applications?view=instructor` → 강사 본인 지원 내역만 반환

### B-3. 수입 / 정산 (taxSection)
- [ ] **이달의 예상 수입 카드**: 총 수업 횟수 × 시급 = 예상 수입
- [ ] **원천징수 자동 계산**: 수입 × 3.3% 표시
- [ ] **순수령액 표시**: 수입 - 원천징수
- [ ] **환급금 조회 버튼**: 연간 납부 원천징수 합계 표시 (mock → 실데이터)
- [ ] **월별 정산 이력 리스트**: 월 / 총수입 / 원천징수 / 순수령액

**백엔드**: `GET /dashboard/instructor-income` 신규 → 승인된 지원 기반 수입 집계

### B-4. 네트워크 보너스 (networkSection - 강사 뷰)
- [ ] 내 레벨 / 이달 PV / 누적 PV 표시
- [ ] 추천인 보너스 이력 리스트 (날짜 / 유형 / 금액)
- [ ] 하위 추천인 수 카운트

### B-5. 대시보드 (dashboardSection - 강사 뷰)
- [ ] 요약 카드: 활성 지원 수 / 승인된 수업 수 / 이달 예상 수입 / 미확인 알림
- [ ] 최근 지원 활동 피드
- [ ] 긴급 공고 알림 배너 (urgent 공고 존재 시)

**완료 기준**: 강사 계정으로 공고 검색 → 지원 → 지원 현황 확인 → 예상 수입 확인까지 단일 세션 완료

## Sprint C. 기관(학교) 기능 완성 (2주)

**목표**: 기관 운영자가 공고를 등록하고 → 지원자를 심사하고 → 배정 확정할 수 있음

### C-1. 공고 관리 (postingsSection / createPostingSection - 기관 뷰)
- [ ] **공고 등록 폼 강화**: 제목 / 과목 / 지역 / 시급 / 수업 시간 / 자격요건 / 마감일 / 긴급여부
- [ ] **공고 목록 - 기관 전용 뷰**: 내 기관 공고만 표시 (타 기관 공고 숨김)
- [ ] **공고 상태 표시**: 모집중 / 마감 / 배정완료 배지
- [ ] **공고 수정/마감 처리**: 상태 변경 버튼

**백엔드**: `POST /postings` → `deadline`, `urgent`, `requirements` 필드 추가 / `GET /postings?owner_only=true`

### C-2. 지원자 심사 (applicationsSection - 기관 뷰)
- [ ] 공고별 지원자 리스트 (공고명으로 그룹화)
- [ ] 지원자 프로필 카드: 이름 / 역할 / 지원일
- [ ] **승인 / 거절 버튼** (각 지원자별)
- [ ] 거절 사유 입력 옵션 (선택)
- [ ] 배정 확정 시 "배정 완료" 상태로 공고 자동 전환

**백엔드**: `GET /applications?posting_owner=true` → 내 공고에 대한 지원자 목록 / `PUT /applications/:id/status` (사유 필드 추가)

### C-3. 정산 현황 (taxSection - 기관 뷰)
- [ ] 배정된 강사별 지급 내역: 이름 / 수업 수 / 시급 / 총 지급액
- [ ] 월별 총 지급 금액 집계 카드
- [ ] "정산 승인" 버튼 → 승인 완료 상태 변경

**백엔드**: `GET /postings/settlement-summary` 신규

### C-4. 대시보드 (dashboardSection - 기관 뷰)
- [ ] 요약 카드: 활성 공고 수 / 총 지원자 수 / 배정 완료 수 / 이달 지급 예정액
- [ ] 최근 지원자 활동 피드

**완료 기준**: 기관 계정으로 공고 등록 → 지원자 승인 → 배정 확정 → 정산 내역 확인까지 완료

## 기술 구현 원칙

1. **프론트엔드**: 단일 파일 SPA 유지, 역할별 분기는 `currentUser.role` 조건으로 처리
2. **백엔드**: `app_jwt.py` 단일 파일 확장, 신규 엔드포인트는 기존 패턴 그대로 추가
3. **Mock 우선**: 백엔드 API 없는 기능은 mock 데이터로 UI 먼저 완성, 이후 API 연결
4. **섹션 재사용**: 동일 섹션을 역할별로 다르게 렌더링 (새 HTML 섹션 추가 최소화)
