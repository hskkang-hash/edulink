# EduLink 단계별 개발 진행표

Last Updated: 2026-03-23

## 진행 원칙
- 개발은 단계별로 진행한다.
- 각 단계는 완료 기준(화면/동작/상태 동기화)을 충족하면 완료로 표시한다.
- 다음 단계 착수 전, 현재 단계 완료 상태를 문서에 반영한다.

## 전체 개발 내역 (Stage Checklist)
- [x] Stage 1. 데모 중심에서 제품 설계 중심으로 방향 전환
- [x] Stage 2. Product Build Plan 재정렬 및 설계 우선 로드맵 반영
- [x] Stage 3. 역할 기반 화면 구조(아키텍처/워크스페이스/갤러리) 구축
- [x] Stage 4. 역할별 상세 프리뷰(학교/교육청/운영자) 구현
- [x] Stage 5. 카드 클릭 네비게이션 및 상단 활성 화면 상태 동기화
- [x] Stage 6. 독립 섹션(학교/교육청/운영자) 생성 및 렌더 체인 연결
- [x] Stage 7. 독립 섹션 카드 -> 실제 제품 화면 이동 타깃 연결
- [x] Stage 8. 카드별 프리셋(필터/정렬/폼 모드) 자동 적용 연결
- [x] Stage 9. 활성 화면 상태 패널에 프리셋 컨텍스트 노출
- [x] Stage 10. 프리셋 해제 액션(원클릭 리셋) 추가
- [x] Stage 11. 프리셋별 핵심 액션 자동 1회 실행(옵션)
- [x] Stage 12. QA/카피/인터랙션 폴리시 최종 점검

## 단계별 완료 기준 (Definition of Done)
- Stage 1-2: 계획 문서가 제품 관점으로 통일되고, 역할/화면 체계가 설계 우선 원칙으로 정리됨
- Stage 3-4: 역할별 주요 화면군이 UI로 확인 가능하고, 사용자 관점 내비게이션이 가능함
- Stage 5-7: 카드 클릭 시 목표 화면 이동 및 상단 상태가 일관되게 동기화됨
- Stage 8-9: 카드 목적에 맞는 초기 상태(필터/정렬/폼)가 자동 적용되고, 현재 프리셋이 상태 패널에 표시됨
- Stage 10-12: 프리셋 제어 완성, 주요 플로우 QA 완료, 용어/메시지/상호작용 마감

## 최근 반영 요약
- 독립 섹션 렌더 연결 완료
- 독립 카드 타깃을 실제 제품 섹션으로 연결 완료
- 카드별 프리셋 적용 로직 추가 완료
- 활성 화면 상태 패널에 프리셋 컨텍스트 표시 완료
- 프리셋 원클릭 해제 버튼 및 상태칩 표시 완료
- 프리셋별 핵심 액션 자동 1회 실행(중복 방지) 완료
- 단계 체크리스트 Stage 1-12 전체 완료 반영
