# Edulink 확장 개발 계획

## 1. 학생/학부모 스토리보드 상세화

### 1.1 사용자 페르소나
- 학부모 A (바쁜 직장인): 주당 수업 관리, 과제 피드백, 비용/정산 한눈 보기
- 학생 B (초중고생): 진도 확인, 출석 체크, 숙제 알림, 강사 후기
- 대리인/보호자: 결제 이력, 안전(강사 범죄 경력), 시간표 조율

### 1.2 주요 화면 & 플로우
- `/student/home`: 오늘 수업, 미완료 과제, 강사 메시지, 이번 달 비용
- `/student/schedule`: 캘린더 + 출석 버튼 + 지각/결석 사유 입력
- `/student/class/:id`: 수업자료, 과제(제출/리뷰), 녹화영상, 실시간 Q&A 링크
- `/student/feedback`: 주간/월간 성취 리포트 + 강사 평점 + 개선 코멘트
- `/student/billing`: 학부모 결제 승인, 회계내역, 환불/취소 조회
- `/student/trust`: 강사 검증 프로필, 범죄경력 + 서류/자격증 + 실사용 리뷰

### 1.3 UX 구조
- 카드 기반 진행 상태 / 다음 액션
- 알림: LMS/푸시 + SMS (미출석, 과제 미완료, 환급 상태)
- 강사 평가 자동 반영 -> 매칭 가중치
- 강사 앱과 기관 앱 연결 흐름

### 1.4 KPI
- 매칭 성공률
- 출석률
- 학습완료율
- 환급 탐색/청구 완료자 비율

## 2. 다단계 알고리즘 재확인 및 적용

### 2.1 법적 요건 (방판법)
- 후원수당 상한: 재화 등 가격 총액의 35% 이내
- 정산 보고, 원장 보관 5년
- PV/BV 모두 후원수당 산정 대상

### 2.2 적용 모델
- 실판매액 `Γ = SUM(price)`
- 후원수당 총액 `C` ≤ `0.35 * Γ`
- PV/BV 가중치
- 21% 리더십 보너스 조건

### 2.3 알고리즘(계산)
1. `base_price`, `pv`, `bv` 기록
2. `Γ` 계산
3. 후원수당 레벨별 비율 (예: 레벨1 3~7%, 레벨2 1~3%, 레벨3 0.5~1%)
4. `C > 0.35 Γ` 시 보정
5. 35% 컴플라이언스 API 제공

### 2.4 모니터링
- 실시간 35% 지표
- 일별/월별 재계산
- 관리자 룰 엔진 + 감사 로그

## 3. HomeTax/삼쩜삼 API 요건

### 3.1 대상 시나리오
- 강사(개인사업자) 세금 신고 및 환급
- 기한 후 신고 미수령 환급
- 3.3% 원천징수 자동 정산
- CMS 후불 이용료 청구

### 3.2 세무 데이터 흐름
1. 강사 API 등록
2. 매출 트래킹
3. 정산 : net = sale*(1-0.033)
4. 홈택스/삼쩜삼 조회/신청
5. 환급 완료 후 자동 정산
6. 이용료 청구

### 3.3 API 명세
- `POST /api/tax/onboard`
- `GET /api/tax/estimate`
- `POST /api/tax/submit`
- `GET /api/tax/status/:submissionId`
- `GET /api/tax/report/:year`
- `POST /api/cms/charge`

### 3.4 데이터 모델
- TaxProfile
- TaxRecord
- TaxClaim
- TaxEvent

### 3.5 스케줄링
- 월말/5월 정기 신고 알림
- 홈택스 간편조회 + 대리인 동의 최소화
- 삼쩜삼 후불 수수료(환급의 20~25%)
