## 1. 제품 개요 및 비전

### 1.1 Product Vision: 프리랜서 강사 통합 운영 인프라

EDULINK는 방과후학교 및 2026년 전 학년으로 확대되는 '늘봄학교' 등 공공/사교육 시장의 **"프리랜서 강사 통합 운영 인프라(Operational Infrastructure)"**입니다. 

기존의 파편화된 매칭, 학사 관리, 세무 정산 시스템을 하나로 통합하고, 다단계 보상 구조(암웨이 모델)와 자동 세금 환급(삼쩜삼 모델)을 결합하여 강사들의 강력한 플랫폼 락인(Lock-in)을 유도합니다.

**핵심 차별화 요소:**
1. **파편화된 시스템 통합**: 매칭, 학사관리, 세무정산을 한 곳에서 관리
2. **다단계 보상 구조**: 암웨이-스타일 MLM으로 강사 수익 극대화
3. **자동 세금 환급**: 삼쩜삼 연동으로 세무 진입장벽 제거
4. **B2G 교육 행정 자동화**: 교육청 통합 모니터링
5. **맞춤형 핀테크**: 프리랜서 대출 및 선정산 서비스

## 3. 핵심 비즈니스 모델 ① : 다단계(네트워크 마케팅) 보상 구조

### 3.1 후원수당 기반 보상 체계

**PV/BV (Point Value / Brand Value) 정의:**
```
PV (Point Value): 
  플랫폼 내 매칭 수수료 또는 자체 교육 콘텐츠 
  (교재, 메타버스/AI 프로그램) 판매액에 부여하는 판매 점수치

BV (Brand Value):
  판매 가격 기반 추가 가치치
```

**후원수당(보너스) 차등 지급:**
```
레벨 1: 20만 PV 이상      → 3% 후원수당
레벨 2: 500만 PV 이상     → 6% 후원수당
레벨 3: 1,000만 PV 이상   → 21% 후원수당

본인 및 하위 그룹 합산 PV 기준
```

### 3.2 독립 그룹 및 리더십 보너스

**조건:**
- 하위 레그(다운라인)가 21% 실적을 달성하여 독립
- 상위 후원자에게 **리더십 보너스** + **직급(Pin)** 부여

**효과:**
- 강사 네트워크 수평 확장 유도
- 상위 강사의 추천 인센티브 극대화

### 3.3 방판법(방문판매법) 컴플라이언스

**엄격한 상한선:**
```
후원수당 총액(C) ≤ 재화 등 가격 합계액(Γ) × 35%
```

**관리 방법:**
- 실시간 35% 규제 비율 모니터링 (월 1회 이상)
- 초과 시 자동 보정 로직 적용
- 감사 로그 기록 (최소 5년 보관)
- Super Admin Web에서 모든 지급 현황 투명 공개

## 4. 핵심 비즈니스 모델 ② : 세무 및 정산 자동화 (삼쩜삼 고도화 모델)

### 4.1 자동 정산 로직

**실시간 순수령액(Net) 산출:**
```
월별 총수익 = SUM(수업료)
원천징수(WH) = 월별 총수익 × 3.3%
순수령액(Net) = 월별 총수익 - 원천징수

예시)
총수익: 1,000,000원
원천징수: 33,000원 (3.3%)
순수령액: 967,000원 → 명세서 제공
```

**자동화 방식:**
- 매 수업 완료 후 자동 집계
- 월말 최종 정산액 도출
- 강사에게 즉시 명세서 제공 (앱 대시보드)

### 4.2 미수령 환급금 원스톱 조회

**간편 프로세스:**
1. 강사가 앱에서 "환급금 조회" 버튼 클릭
2. 카카오톡/PASS 간편인증 → 홈택스 데이터 자동 연동
3. **예상 환급액** 무료 즉시 조회
   - 세무대리인 수임동의 절차 없음 (진입 장벽 극소화)
   - 신뢰도 높은 정부DB 활용

### 4.3 기한 후 신고 지원 (5년 소급)

**대상:**
- 최근 5년간 신고하지 않아 돌려받지 못한 미수령 환급금
- 매년 5월 정기신고 외 통년 기한 후 신고 가능

**지원 내용:**
- 앱 내에서 "기한 후 신고" 버튼 별도 제공
- 과거 연도별 소득 기록 자동 수집
- 홈택스/삼쩜삼 플랫폼 연동으로 원스톱 신고

### 4.4 이용료 후불 결제 (CMS 자동 출금)

**비용 구조:**
```
예상 세액 조회: 무료
신고 대행료: 조회 후 유료 (선택)
환급금 수령 후 이용료 자동 출금 (CMS)

이용료 과금 타이밍:
- 실제 환급금을 수령한 후 (또는)
- 신고 완료 4개월 이후

이용료 비율: 환급액의 20~25% (시장 표준)
```

**고객 경험:**
- 예상 환급액 조회 → 후불 결제 동의 → 환급금 입금 후 자동 차감
- 앱에서 환급 상태, 이용료 예상액, 차감액 실시간 추적

## 5. 강사 매칭 및 운영 솔루션 (차별화 인프라)

### 5.1 긴급 매칭 (SOS 알림)

**시나리오:**
```
학교 현장에서 강사 결원 발생
         ↓
수요기관(학교)이 "긴급 출강 SOS" 공고 등록
         ↓
조건(과목, 지역, 시간)에 맞는 강사 모두에게
실시간 푸시/SMS 알림 발송
         ↓
강사가 앱에서 즉시 수락/거절
         ↓
가장 먼저 수락한 강사에게 배정 → 즉시 정산 예약
```

**효과:**
- 학교: 30분 내 결원 해결
- 강사: 긴급 수입 기회 (수수료 할증 가능)

### 5.2 양방향 일정 예약 (온디맨드)

**강사 관점:**
```
강사 캘린더에 Available Slots(수업 가능 시간) 설정
→ "수요일 18~19시 영어 가능" 등 공개
```

**기관 관점:**
```
기관이 강사의 가용 시간을 실시간 조회
→ 원하는 시간대 직접 픽업 (즉시 예약)
→ 강사는 알림으로 수락/거절 선택
```

**장점:**
- 강사의 시간 활용도 최대화
- 기관의 예약 편의성 증대
- 조율 시간 단축

### 5.3 에스크로 기반 안전 결제

**결제 흐름:**
```
1. 수요기관(학교)이 PG사(Stripe/NHN KCP)를 통해
   사전 결제(에스크로)

2. 수업료는 에스크로 계정에 보관

3. 강사의 수업 완료 확인 후
   3.3% 원천징수액 공제 → 강사 계좌로 자동 정산

4. 결제 취소 시
   에스크로에서 기관으로 자동 환불 
```

**안전성:**
- 강사: 수업료 미정산 위험 제거
- 기관: 강사 노쇼 시 환불 보장
- 플랫폼: 결제 분실 없음 (신용도 향상)

## 8. 🚀 향후 로드맵 (Post-MVP 개선 사항)

### Phase ② (Month 2-3): AI 매칭 + 질의응답

**기능:**
1. **내신 성적 기반 매칭**: AI가 학생 성적 분포 분석 → 강사 추천
2. **AI 질의응답**: 강사 부재 시 간단한 질문 자동 답변
3. **매칭 성공률 예측**: 과거 데이터 기반 무의식적 바이어스 제거
4. **강사 커리큘럼 최적화**: 학생별 학습 속도 맞춤 제안

**기술:**
- TensorFlow / PyTorch
- Natural Language Processing (NLP)
- Collaborative Filtering

---

### Phase ③ (Month 4-5): 데이터 기반 핀테크 확장

**기능:**
1. **강사 대체 신용평가 모델**: 수업 횟수, 정산 이력, 평점 → 신용도
2. **선정산(Advance Pay)**: 신용도 높은 강사 월 정산 전 사전 정산
3. **프리랜서 대출**: 신용도 기반 마이크로 대출 (금리 6~12%)
4. **신용 포인트**: 대출 상환 시 포인트 축적 → 한도 증가

**추정 수익:**
- 선정산 수수료: 정산액의 1~2%
- 대출 이자: 연 10%
- 월 대출 규모: 1억 원대

---

## 10. 비기능 요구사항

### 10.1 성능 (Performance)

| 지표 | 목표 |
|------|------|
| 앱 시작 | < 2초 |
| 홈 로딩 | < 1.5초 |
| API 응답 | < 500ms (p99) |
| 인터랙션 | < 100ms |
| 이미지 | < 100KB |
| 동시 접속 | 10,000 MAU |

### 10.2 보안 (Security)

**인증:** OAuth 2.0 (카카오톡/PASS), JWT (1시간 만료)

**암호화:** HTTPS TLS 1.3, AES-256

**결제:** PCI-DSS 3.2.1, 3D Secure

**개인정보:** GDPR/개인정보보호법, PII 마스킹

**범죄경력:** 정부 공식 DB, SSL/TLS, 감사 추적

### 10.3 확장성 (Scalability)

- **마이크로서비스**: 독립 배포/스케일
- **DB**: 마스터-슬레이브, 파티셔닝
- **캐싱**: Redis, CDN (Cloudflare)
- **배포**: Kubernetes, 자동 스케일링, 무중단 배포

### 10.4 신뢰성 (Reliability)

- **가용성**: 99.9% SLA
- **멀티 데이터센터**: 서울, 부산
- **백업**: 일 1회 (증분), 지역 복제 (RPO: 1시간)
- **모니터링**: DataDog, Sentry, CloudWatch

### 10.5 준수 (Compliance)

- **방판법**: 35% 모니터링, 5년 보관
- **세법**: 3.3% 원천징수, 연말정산 보고
- **통신판매법**: 환불/취소 정책, 정보공시
- **개인정보보호법**: 동의 관리, 삭제권, 진단

## 12. 용어 정의

| 용어 | 정의 |
|------|------|
| **PV** | 플랫폼 매칭 수수료 또는 교육 콘텐츠 판매액 기반 점수치 |
| **BV** | 판매 가격 기반 추가 가치치 |
| **후원수당** | 본인 및 하위 그룹 PV 합계에 따라 지급하는 MLM 보너스 (3%~21%) |
